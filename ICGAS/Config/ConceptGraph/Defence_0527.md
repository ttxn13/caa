# 概念标准化示例-航母编队防空反导场景（AJP-5 关系重构版）
## 一、版本说明

本文件以用户给定的 `E_NODE_TYPE` 与 `E_EDGE_TYPE` 为约束，对既有 Defence 场景进行重新组织。`DEC` 在本文中表示 `Decisive Condition`，即“决定性条件”，不表示 `Decision Point`。由于当前边定义采用 `DEC_TO_EFT`、`EFT_TO_ACT`、`ACT_TO_TSK`、`TSK_TO_PLT` 的方向，本文将其解释为从目标条件向效果、行动、任务和平台资源的规划分解关系。执行评估时可按相反方向汇总完成度。
| 字段 | 内容 |
| --- | --- |
| 场景编号 | Defence |
| 场景名称 | 航母编队防空反导场景 |
| 场景类型 | DEFENSE |
| Schema版本 | 3.0-ajp5-aligned-user-edge-set |
| 节点类型 | MIS, TSK, ACT, OBJ, COG, DEC, EFT, LOO, COA, PLT |
| 边类型 | MIS_TO_COG, MIS_TO_OBJ, OBJ_TO_LOO, LOO_TO_COA, COG_TO_DEC, OBJ_TO_DEC, DEC_TO_DEC, DEC_TO_EFT, EFT_TO_ACT, ACT_TO_ACT, ACT_TO_TSK, TSK_TO_PLT |

## 二、上级使命

| 节点ID | 名称 | 说明 | 附加属性 |
| --- | --- | --- | --- |
| MIS-1 | 保障航母编队生存和持续行动能力 | 在敌空袭、反舰导弹突防、电子干扰和侦察引导威胁条件下，保持航母编队核心单元安全、持续空情认知、防空反导链路和动态重规划能力。 |  |

## 三、作战目标

| 节点ID | 名称 | 说明 | 附加属性 |
| --- | --- | --- | --- |
| OBJ-1 | 保持航母及核心护航舰艇安全 | 确保航母和关键护航舰艇不遭受重大毁伤，维持编队基本作战能力。 |  |
| OBJ-2 | 建立连续可信的空情态势 | 持续掌握敌空中目标、导弹目标、干扰目标的分布、航迹、威胁等级和行动趋势。 |  |
| OBJ-3 | 削弱敌空中突击链效能 | 降低敌侦察定位、电子干扰、协同攻击和导弹齐射能力。 |  |
| OBJ-4 | 形成多层接续防空反导能力 | 建立外层截击、中程拦截、近程防御和末端补防的连续处置链。 |  |
| OBJ-5 | 保持指挥控制和动态重规划能力 | 在敌情变化、我方节点受损或资源消耗异常时保持方案调整能力。 |  |

## 四、作战重心

| 节点ID | 名称 | 说明 | 附加属性 |
| --- | --- | --- | --- |
| COG-1 | 航母生存与规避机动能力 | 我方核心防护重心，航母作为编队核心平台，其生存能力直接决定防御任务成败。 | own |
| COG-2 | 护航舰艇防护阵位保持能力 | 我方防护体系重心，护航舰艇阵位决定防护扇面、拦截通道和末端补防能力。 | own |
| COG-3 | 远程预警探测能力 | 我方感知重心，预警机、舰载雷达和外部情报源构成持续空情态势基础。 | own |
| COG-4 | 态势融合与威胁识别能力 | 我方认知重心，目标分群、威胁排序和趋势判断支撑火力分配。 | own |
| COG-5 | 敌侦察引导链 | 敌方体系重心，支撑敌方发现、定位、引导和协同攻击。 | enemy |
| COG-6 | 敌电子干扰链 | 敌方体系重心，削弱我方探测、通信和指控链路稳定性。 | enemy |
| COG-7 | 外中层防空拦截能力 | 我方火力处置重心，支撑外层截击和中程区域防空接续。 | own |
| COG-8 | 近程点防御与漏网目标处置能力 | 我方末端防御重心，支撑饱和攻击条件下的最后防护。 | own |
| COG-9 | 指挥控制链路保持能力 | 我方指控重心，支撑态势共享、指令下达、资源调度和跨平台协同。 | own |
| COG-10 | 动态重规划与资源再分配能力 | 我方适应性重心，支撑敌情变化和资源损耗条件下的方案修正。 | own |

## 五、决定性条件

| 节点ID | 名称 | 说明 | 附加属性 |
| --- | --- | --- | --- |
| DEC-1 | 航母重点防护态势建立 | 航母处于可接受风险范围内，重点防护扇面与规避机动方案保持有效。 |  |
| DEC-2 | 护航舰艇防护阵位保持有效 | 护航舰艇能够持续承担探测、拦截和防护任务。 |  |
| DEC-3 | 关键威胁方向形成连续预警覆盖 | 主要威胁方向持续被预警探测和稳定跟踪。 |  |
| DEC-4 | 敌主攻分群完成识别与威胁排序 | 敌主攻分群、佯攻分群和干扰分群被区分并形成处置优先级。 |  |
| DEC-5 | 敌侦察引导节点效能下降 | 敌目标定位连续性下降，攻击协同精度降低。 |  |
| DEC-6 | 敌电子干扰平台被压制或驱离 | 我方探测、跟踪和通信链路受干扰程度降低。 |  |
| DEC-7 | 外层拦截窗口建立 | 敌攻击平台或高威胁目标在远端处置窗口内被削弱。 |  |
| DEC-8 | 中程区域防空火力完成接续准备 | 来袭目标进入中程拦截链并完成火力分配。 |  |
| DEC-9 | 近程点防御链路闭合 | 漏网目标被重新捕获并进入末端处置流程。 |  |
| DEC-10 | 指挥控制链路保持可用 | 态势共享、指令下达和资源调度流程保持连续。 |  |
| DEC-11 | 触发重规划条件被及时识别 | 系统能够依据敌情变化、资源消耗或条件偏差生成修正方案。 |  |

## 六、作战效果

| 节点ID | 名称 | 说明 | 附加属性 |
| --- | --- | --- | --- |
| EFT-1 | 远程预警探测链路建立 | 预警机、舰载雷达或外部情报源形成早期发现能力。 |  |
| EFT-2 | 海基区域搜索形成稳定补充航迹 | 护航舰艇对主要威胁方向完成航迹补充和持续跟踪。 |  |
| EFT-3 | 敌主攻分群、佯攻分群和干扰分群被区分并排序 | 威胁分群识别形成主攻方向、佯攻方向和干扰方向判别结果。 |  |
| EFT-4 | 态势共享、指令下达和资源调度流程保持连续 | 探测、跟踪、火力分配、指令下达和评估反馈的信息闭环保持可用。 |  |
| EFT-5 | 敌侦察定位连续性下降且电子干扰影响减弱 | 电子压制、诱骗或驱离行动降低敌侦察引导和电子干扰效能。 |  |
| EFT-6 | 敌攻击平台或高威胁空中目标被前出截击削弱 | 舰载战斗机在远端对敌攻击平台或高威胁编队实施截击。 |  |
| EFT-7 | 远程舰空导弹建立外层拦截窗口 | 区域防空舰远程舰空导弹对高威胁目标形成外层处置条件。 |  |
| EFT-8 | 来袭目标进入中程拦截链并完成火力分配 | 中程区域防空体系完成接续拦截准备和火力分配。 |  |
| EFT-9 | 漏网目标被重新捕获并进入末端处置流程 | 近程点防御、末端火力和软杀伤措施进入闭合处置链。 |  |
| EFT-10 | 航母受击风险被控制在可接受范围内 | 航母规避机动和护航阵位调整使核心单元风险降低。 |  |
| EFT-11 | 拦截效果、残余威胁和资源消耗被评估 | 拦截效果评估形成残余威胁、毁伤结果和武器消耗判据。 |  |
| EFT-12 | 系统根据敌情变化或我方资源变化生成修正方案 | 动态重规划形成后续行动、资源再分配和方案调整结果。 |  |

## 七、可行动作

| 节点ID | 名称 | 说明 | 附加属性 |
| --- | --- | --- | --- |
| ACT-1 | 前出空情探测 | 预警机、舰载雷达和外部情报源前出建立早期预警与远程探测链路。 |  |
| ACT-2 | 海基区域搜索 | 护航舰艇对主要威胁方向实施区域搜索、航迹补充和目标持续跟踪。 |  |
| ACT-3 | 威胁分群识别 | 对敌空袭编队、导弹群和干扰平台进行分群、意图识别和威胁排序。 |  |
| ACT-4 | 态势共享与链路保持 | 保持探测、跟踪、指令下达、火力分配和评估反馈的信息闭环。 |  |
| ACT-5 | 电子压制与诱骗 | 对敌侦察引导节点和电子干扰平台实施干扰、压制、诱骗或驱离。 |  |
| ACT-6 | 舰载机前出截击 | 使用舰载战斗机对敌攻击平台、高威胁编队或突防目标进行前出拦截。 |  |
| ACT-7 | 远程舰空导弹拦截 | 使用区域防空舰远程舰空导弹建立外层拦截窗口。 |  |
| ACT-8 | 中程区域防空接续 | 对进入中程拦截窗口的来袭导弹和空中目标进行火力分配与接续拦截。 |  |
| ACT-9 | 近程点防御与软杀伤 | 对漏网目标实施末端点防御、近程火力拦截、电子干扰和诱偏处置。 |  |
| ACT-10 | 航母规避与护航阵位调整 | 调整航母规避机动和护航舰艇防护阵位，保持重点防护扇面。 |  |
| ACT-11 | 拦截效果评估 | 对已拦截目标、漏网目标、毁伤效果和武器消耗进行综合评估。 |  |
| ACT-12 | 动态重规划 | 根据敌情变化、资源损耗、链路状态和防御效果生成修正方案。 |  |

## 八、关键任务

| 节点ID | 名称 | 说明 | 附加属性 |
| --- | --- | --- | --- |
| TSK-1 | 建立远程预警与空情监视 | 通过舰载预警机、舰载雷达和外部情报源形成持续空情认知。 |  |
| TSK-2 | 识别敌方主攻方向和威胁分群 | 对敌机群、导弹群和干扰平台进行分群、意图识别和威胁排序。 |  |
| TSK-3 | 保护航母及关键护航舰艇 | 明确航母为最高防护对象，组织护航舰艇、舰载机和防空火力形成重点防护。 |  |
| TSK-4 | 维持编队指挥控制与数据链稳定 | 保证探测、跟踪、火力分配和拦截评估之间的信息闭环连续。 |  |
| TSK-5 | 建立多层防空反导拦截体系 | 形成外层截击、中层区域防空、近程点防御和软杀伤配合。 |  |
| TSK-6 | 压制或迟滞敌侦察引导与电子干扰能力 | 削弱敌方发现、跟踪、引导和协同攻击能力。 |  |
| TSK-7 | 组织拦截后评估与再分配 | 评估拦截效果，发现漏网目标后快速补充拦截资源。 |  |
| TSK-8 | 保持方案动态调整能力 | 当敌主攻方向变化、我方资源受损或决定性条件未达成时进行重规划。 |  |

## 十三、一致性校验规则

| 规则编号 | 规则 |
| --- | --- |
| R-1 | 场景必须存在且仅存在一个 MIS 节点。 |
| R-2 | 每个 OBJ 必须由 MIS_TO_OBJ 连接到 MIS-1。 |
| R-3 | 每个 COG 必须由 MIS_TO_COG 连接到 MIS-1，并至少牵引一个 DEC。 |
| R-4 | 每个 OBJ 必须通过 OBJ_TO_LOO 对应一条 LOO。 |
| R-5 | 每个 LOO 可通过 LOO_TO_COA 支撑一个或多个 COA；sequence 表示该 LOO 在该 COA 中的阶段顺序。 |
| R-6 | 每个 DEC 必须由 OBJ_TO_DEC 连接到至少一个 OBJ，并由 DEC_TO_EFT 连接到至少一个 EFT。 |
| R-7 | 每个 EFT 必须存在且仅存在一个 EFT_TO_ACT 出边，以满足当前枚举中 EFT_TO_ACT 的 1->1 约束。 |
| R-8 | ACT_TO_ACT 和 DEC_TO_DEC 仅表达基准流程顺序，不表达严格物理因果或唯一战术顺序。 |
| R-9 | 每个 TSK 必须通过 TSK_TO_PLT 连接至少一个可用平台。 |
| R-10 | 所有边的起始节点和终止节点必须存在，所有节点 ID 与边 ID 必须唯一。 |

## 十四、工程使用说明

当前图结构采用“规划分解边”为主。其核心链路为 `MIS_TO_OBJ / MIS_TO_COG → OBJ_TO_LOO / OBJ_TO_DEC / COG_TO_DEC → DEC_TO_EFT → EFT_TO_ACT → ACT_TO_TSK → TSK_TO_PLT`。`LOO_TO_COA` 用于表达多条作战线共同构成一个候选行动方案，其中 `sequence` 表示该作战线在候选方案中的阶段顺序。`DEC_TO_DEC` 与 `ACT_TO_ACT` 表示基准流程顺序，不应在算法中解释为不可改变的唯一因果链。
